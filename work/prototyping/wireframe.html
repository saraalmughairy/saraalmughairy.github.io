Interactive Wireframe

Create a wireframe prototype for an app for per owners.

To develop these wireframes, I first sketched general layout ideas by hand, annotating the flow from one screen to the next. Once I had established the order and content of each screen, I produced basic renderings in Adobe Illustrator, eschewing cosmetic details in keeping with the function of a wireframe. For the typeface, I chose helvetica, regular weight, for its universal readability and generic, unobtrusive tone. Graphic elements like navigation bars, buttons, and text boxes were limited to simple rectangles, and color was limited to greyscale (except for the data, where color is necessary for clear visualization). After exporting .PNG images for each screen, I used Invision to build the interactivity between pages. Invision uses a simple interface to connect static images through the creation of hotspots- areas that when tapped, clicked, or gestured upon trigger another page.

Reflection

In some senses, this prototyping method is limited (i.e. transitions are limited, less customizable and interactive than coding), but it is quite capable for demonstrating navigation and interactions, which was the goal of this particular process.
