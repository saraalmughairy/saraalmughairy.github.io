problem space.


how can we help businesses connect with their data?

Business Intelligence tools typically help companies to retrieve, analyze, and transform their data with the goal of unconvering actionable insights.

No doubt, BI has major potential.

Billions of dollars are spent by companies per year indicate the perceived value of business intelligence.

opportunity.


Ease tension between usability and customizability.

Typically, the more customizable a BI solution is, the more it requires expertise. How can we lower the barrier for entry while maintaining customizability?


1. BI tools are often difficult to use.

When Matillion asked over 10,000 executives and decision-makers, ‘What is your biggest management information challenge?’ They found that a whopping 36% of respondents felt their BI tools did not adequately support self-service. Steep learning curves commonly result in bottlenecks where only key individuals within the company know how to access the data.


2. BI tools often are not sufficiently personalizable.

In the Eckerson Tech Target Survey, 67% of respondents said the biggest challenge in utilizing BI tools is customizing them to meet business requirements. So much so that 39% build BI functionality from scratch rather than relying on commercial software.

solution.


A desktop app that enables businesses to stay informed of critical metrics through live dashboards.

features.


1. Extract multi-dimensional data slices.

Access information from your company’s local databases. Then filter the data via SQL queries and save slices with a memorable name.

2. Visualize data using D3.

After slicing the data, the user is taken to the Explorer page, which is home to the D3 chart creation tools. A control panel allows non-D3 users to easily make charts and a code sandbox allows advanced users full D3 functionality. Once a chart is created, it can be saved to any dashboard.

3. Create Real-Time Dashboards.

Once a D3 chart is created, multiple charts can be combined onto a dashboard. Charts exist on drag-and-drop, resizable cards making the dashboard entirely configurable. Charts are updated in real-time as the user's local databases are updated. Dashboards let users track the current status of metrics and key performance indicators for their business.

4. Share, Embed, and Export Charts.

D3 visualations can be exported as a non-proprietery SVG object or pdf. By using a firebase database, we are able to remotely keep track of all user accounts. So when a user shares a chart with coworkers, they receive a notification on their desktops.

technologies.


What makes dataLab special? The way it's built.
1. Full Javascript Stack

First off, dataLab was created using JavaScript, the most widely used programming language on the web. Node.js, a platform built on Chrome's JavaScript runtime, allows Javascript to run server-side, positioning JavaScript as the only language that runs in both the front- and back-end. The sheer popularity of Javascript means that companies will be able to easily find someone internally or externally to customize dataLab to suit their specific needs.


2. Desktop Application

For the scope of this project, we decided to create a desktop application versus a web-based app. This means that businesses can use dataLab with their local databases rather than host their sensitive data on a cloud. We used Electron to create the desktop application because it allowed us to make a native app, but with web technologies like JavaScript, HTML, CSS, React, and Redux. Importantly, this also means dataLab can easily be converted to a web application, should the need arise.


3. D3 Data Visualizations

In addition to being beautiful and interactive, D3 has three advantages: First, D3 is customizable. D3 lets you visualize any business insight you may need. Secondly, D3 is executable JavaScript code, which means your visualizations can update in real-time. Third, D3 is heavily documented and has a strong online community, making it easy to source inspiration and help from other D3 users.


4. Authentication

We use Auth0 to handle user authentication and host user accounts through Firebase. Using secure logins lets businesses better control their data by providing different access levels for each user. Auth0 also allows social authentication through Facebook, Google, or any OAuth provider.

secret sauce.


dataLab achieved some awesome feats.

1. React's Virtual DOM.

React is fast because it creates its own virtual DOM, which only updates changes to the DOM rather than re-rendering the entire DOM with every small change. To achieve that, React needs to have control over everything that should happen with the DOM.

2. But, wait! D3 needs the DOM!

At first glance, this means React and D3 are not an inherently good fit. This is because while React is updating a node on the DOM, D3 may be trying to update the same node in a different way.

3. Aha! Use a faux DOM!

We resolved this conflict by pointing D3 at a DOM-impersonator called a “faux-Dom” that it has full control of. D3 nodes from this faux-dom can then be injected into React to render. The beautiful faux-dom library we used was created by Oliver Caldwell.

4. Generating D3 charts from multiple sources of input.

To combine user inputs from the Control Panel and Sandbox, we wrap any custom D3 code a user enters in a function which is then invoked using the settings from the control panel to generate another function. This function is invoked with data from the database as it comes in to keep the visualization updating in real-time.
